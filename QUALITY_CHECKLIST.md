# SafeFlow Thailand — Quality Checklist

Before considering the mockup finished, verify:

## Visual quality

- [ ] Does the app look like a premium consumer product?
- [ ] Does it avoid generic SaaS dashboard style?
- [ ] Are the colors bright but controlled?
- [ ] Does it have a memorable identity?
- [ ] Does the dashboard feel alive?
- [ ] Does the map feel exciting?
- [ ] Are the cards polished?
- [ ] Are buttons satisfying to click?
- [ ] Is typography strong and readable?
- [ ] Is spacing generous?

## UX quality

- [ ] Can the user understand the product in 10 seconds?
- [ ] Are the main CTAs obvious?
- [ ] Is the taxi checker fast and simple?
- [ ] Is the trip planner enjoyable to fill out?
- [ ] Does the itinerary result feel rewarding?
- [ ] Does every result explain “why”?
- [ ] Does the app avoid fear-mongering?
- [ ] Does the app feel personal?

## Motion quality

- [ ] Page transitions are smooth
- [ ] Cards reveal nicely
- [ ] Selected states animate
- [ ] Map pins animate
- [ ] Buttons have good hover states
- [ ] Animations are not annoying
- [ ] The UI does not feel slow

## Product clarity

- [ ] Dashboard shows weather / PM2.5 / travel comfort
- [ ] Taxi checker shows fair range and risk
- [ ] Trip planner includes profile, vibe, food, budget, accessibility
- [ ] Itinerary includes map, timeline, scores, explanation
- [ ] Local business support is visible
- [ ] Risk/friction idea is understandable

## Code quality

- [ ] Components are reusable
- [ ] Mock data is separated
- [ ] No huge single-file mess
- [ ] Naming is clean
- [ ] Styling is consistent
- [ ] Mobile/tablet layout is acceptable
- [ ] README explains concept
